# 網站重構評估 Rubric

> **目標**：把現有網站重構成一個有科技感、設計具個性（非 AI 模板）、且能輕鬆透過 GitHub Pages 部署的靜態網站。

---

## 使用方式

- 每個維度依 **0–4 分** 評分，乘上權重後加總。
- **達成率 = 加權總分 ÷ 最高加權分（136）× 100%**；**達成率 ≥ 80%** 視為達標。
- 單一維度 ≤1 分，視為未通過該項，需修正後再評。
- 評分時對照「4 分標準」自問：「我能說出這個選擇背後的理由嗎？」說不出理由的設計，多半就是模板味。

| 等級 | 分數 | 意義 |
|------|------|------|
| 4 | 優秀 | 刻意、自洽、有觀點 |
| 3 | 良好 | 完成度高，仍有打磨空間 |
| 2 | 及格 | 能用，但平庸或有明顯缺陷 |
| 1 | 不足 | 半成品或方向錯誤 |
| 0 | 缺席 | 完全沒做 |

## 風格參考方向

> 給維度 1、2 評分時的選風格依據。**建議先選一個主風格貫徹**；要混搭也可以，但需有明確主從，否則容易回到「拼貼＝模板感」。每種都附避雷重點，提醒哪裡一不小心就掉回 AI 範本味。

### A. 科技暗黑模式（Trading Terminal 風）
- **調性**：Bloomberg／交易終端機，高資訊密度、專業冷靜
- **配色**：近黑底（#0d0f12 一類）＋ 單一霓虹強調色（漲綠跌紅或琥珀），灰階分層
- **字體**：等寬數字（tabular-nums）是靈魂；標題可用幾何 sans，數據區用 mono
- **圖表**：細格線、深色畫布、發光的線；K 線／權益曲線是主角
- **避雷**：別堆毛玻璃卡片與無理由的紫藍漸層；霓虹色只用一兩個，不是整片發光

### B. 幾何抽象與數據可視化（Data-viz Forward）
- **調性**：把圖表本身當視覺主體，幾何形狀構成版面
- **配色**：中性底＋ 2–3 個飽和對比色作資料編碼，色彩有「意義」而非裝飾
- **字體**：強烈的幾何無襯線（如 Space Grotesk 類），大小對比明顯
- **圖表**：線／面積／散點作為 hero，輔以圓、線段、網格等幾何元素延伸
- **避雷**：圖表要能讀懂、色彩對應資料；別為了好看放假數據或無意義動畫

### C. 極簡主義（Minimalism）
- **調性**：留白主導、克制、讓內容自己說話
- **配色**：單色或近單色＋ 一個重點色；高對比但不刺眼
- **字體**：以排版層次建立秩序，靠字重與間距而非裝飾
- **圖表**：去除多餘格線與邊框，data-ink ratio 高
- **避雷**：極簡 ≠ 空泛；要有刻意的字體與間距節奏，否則就是「乾淨但無聊」的範本

### D. 終端機 / Monospace（CLI 復古）
- **調性**：命令列美學，工程師感、帶點玩味
- **配色**：黑底綠字或琥珀字，或淺底＋ mono
- **字體**：全站等寬（JetBrains Mono、IBM Plex Mono 類），可帶游標／ASCII 元素
- **圖表**：可走 ASCII／點陣風，或讓數字表格成為主視覺
- **避雷**：別只是換 mono 字就當科技；要在互動與細節（輸入框、prompt）貫徹

### E. 瑞士國際主義 / Swiss Grid
- **調性**：嚴謹網格、機構級的秩序與信任感
- **配色**：白／黑為主＋ 紅或單一強調色
- **字體**：Helvetica／Neue Haas 一類的中性 sans，靠網格與層級
- **圖表**：對齊嚴格、標籤清楚、留白規律，適合大量表格與報表
- **避雷**：網格要真的對齊；歪一點點就破功，反而顯得草率

### F. 財經編輯風（FT／Economist 印刷感）
- **調性**：成熟、可讀、有編輯品味，適合內容＋數據並重
- **配色**：米色／鮭紅紙感底色＋ 深墨色文字
- **字體**：襯線標題＋ 無襯線內文，對比鮮明
- **圖表**：印刷級資料圖，註解與來源講究
- **避雷**：別讓襯線只出現在標題卻無整體編輯邏輯；色票要克制

### G. 藍圖 / 工程示意（Blueprint）
- **調性**：技術製圖、示意圖質感，強調「系統與結構」
- **配色**：深藍／墨藍底＋ 青色線條，或反白工程圖風
- **字體**：技術感 sans ＋ mono 標註
- **圖表**：格線、座標、標註線、虛線框營造「圖紙」感
- **避雷**：格線與標註要服務資訊，不是滿版裝飾

## 評分維度

### 1. 設計個性 — 避免「AI 模板感」（權重 ×5）

這是這次重構的核心，獨立成一個維度。

| 分數 | 標準 |
|------|------|
| 4 | 有清楚的設計觀點：自選的字體配對、刻意的色彩系統、非對稱或有節奏的版面；看一眼就知道「這是某個人做的網站」 |
| 3 | 整體有風格，但仍殘留一兩處通用樣式 |
| 2 | 乾淨但無記憶點，換掉文字就像任何一個範本 |
| 1 | 明顯套版：置中 hero + 一句 slogan + 兩顆按鈕 + emoji 卡片 |
| 0 | 直接是預設模板 |

**扣分警訊（AI 模板特徵）**
- 無理由的紫／靛藍漸層、毛玻璃卡片堆疊
- 全站只有 Inter／預設 sans-serif，無字體層次
- 千篇一律的「三欄 emoji 功能卡」
- 所有元素都置中、所有圓角都一樣、所有陰影都一樣
- 內容空泛的佔位文案（「Empower your workflow」這類）

---

### 2. 科技感的視覺語言（權重 ×4）

風格可以多樣（極簡、終端機／monospace、賽博、blueprint、資料視覺化感都可以），但要「成立」。

| 分數 | 標準 |
|------|------|
| 4 | 選定一個科技方向並貫徹：字體、色彩、格線、動效彼此呼應，細節（如等寬數字、grid line、code 樣式）強化主題 |
| 3 | 科技調性明確，少數元素跳脫主題 |
| 2 | 有科技元素但拼貼，風格不統一 |
| 1 | 只是深色背景配亮綠字就當作科技感 |
| 0 | 與科技調性無關 |

|------|------|

| 3 | 大致一致，少數硬編碼數值 |
| 2 | 有重複但散落的樣式 |
| 1 | 各頁各自為政 |
| 0 | 無一致性可言 |

| 分數 | 標準 |

| 3 | 結構合理，仍有少量待整理 |

| 1 | 只是搬移檔案，問題照舊 |

| 2 | 桌面正常但手機體驗勉強 |

| 3 | 大致快，少數未優化資源 |
| 2 | 可接受但有明顯肥大資源 |
| 1 | 載入緩慢 |
| 0 | 嚴重卡頓 |

| 分數 | 標準 |

| 3 | 大致達標，少數缺漏 |
| 2 | 部分處理 |

| 分數 | 標準 |
|------|------|

| 3 | 可部署，需少量手動設定 |
| 2 | 能上線但路徑或 base 有問題 |
| 1 | 部署後破圖／樣式跑掉 |
| 0 | 無法部署 |

### 9. 回測功能（權重 ×4）

因為部署在 GitHub Pages（純靜態、無後端），回測需在**瀏覽器端以 JavaScript 執行**；資料量大時應考慮 Web Worker，避免主執行緒卡頓。

| 分數 | 標準 |
|------|------|
| 4 | 可載入歷史數據並設定策略參數（進出場條件、指標如 MA／ATR／布林通道）；輸出完整績效指標（總報酬、最大回撤 MDD、勝率、夏普值、交易次數）；在 K 線上標記進出場並繪製權益曲線；計算放在 Web Worker 不卡 UI；結果可重現、可匯出 |
| 3 | 回測可運作、績效指標齊全，但缺視覺化或效能優化 |
| 2 | 只能算單一指標或固定策略，無法調參數 |
| 1 | 有雛形但結果不正確或無法操作 |
| 0 | 未實作 |

**正確性警訊**
- 用到未來資料（look-ahead bias）：以當根收盤價當作可在同一根成交的依據
- 未計入手續費／滑價，績效虛高
- 報酬率與最大回撤的計算基準不一致

---

### 10. 多頁架構與分頁管理（權重 ×3）

| 分數 | 標準 |
|------|------|
| 4 | 功能依領域拆頁（如：總覽／個股／回測／設定），各頁職責單一、互不耦合；導覽清楚、URL 有意義且可深連結分享；header／nav 等共用版型不重複；已正確處理靜態站路由（多 HTML 檔，或 SPA 搭配 hash／history mode 並備妥 404 fallback） |
| 3 | 已分頁且導覽合理，路由或共用版型有小瑕疵 |
| 2 | 有分頁但切換生硬、版型重複、URL 無意義 |
| 1 | 名義上分頁，實際內容仍擠在一頁 |
| 0 | 全部擠在單一頁面 |

**GitHub Pages 路由提醒**：若採 SPA history mode，重新整理子路徑會 404，需用 `404.html` 轉址或改用 hash routing；多 HTML 檔則最單純、最穩。
